**版本**: 0.3.2  
**状态**: 待修复

## 1. 编辑器 B 键硬编码关闭

**文件**: `editor/EditorScreen.java` 第 690 行

```java
if (keyCode == 66) { EditorLogger.action(EditorLogger.SCREEN, "CLOSE", "B"); onClose(); return true; }
```

B 键 (keyCode=66) 硬编码为关闭键。用户把 `EDITOR_KEY` 映射到任意键、重置游戏控制后，B 仍会关闭编辑器——因为不是绑定问题，是编辑器内部写死了。

**修复**: 删除第 690 行。只保留 ESC (keyCode=256) 关闭。

---

## 2. 脚本播放时关闭其他 GUI

**需求**: 脚本开始播放时自动关闭所有已打开的 GUI 界面（物品栏、聊天、其他模组界面），避免遮挡画面。

**方案**: 在 `CameraManager.startScriptInternal` 中加入：

```java
Minecraft.getInstance().setScreen(null);
mc.player.closeContainer();
```

执行时机：`startScriptInternal` 第一行，在设置键鼠锁定之前。

---

## 3. 新建脚本串旧脚本状态（letterbox 旧格式残留）

**文件**: `editor/EditorScreen.java` + `script/ScriptParser.java`

**根因**: `onNewScript` 创建 letterbox clip 时仍用旧格式（`enabled/aspect_ratio/fade_in/fade_out`，无 `keyframes`）。`ScriptParser.parseLetterboxClips` 第 358 行用 `requireArray` 强制要求 keyframes——解析失败，CameraManager 维持旧脚本数据，预览/播放仍为旧脚本内容。

**修复 3a** — EditorScreen.java 三处（init firstInit、menuBar onNewScript、leftPanel onNewScript），把旧格式 letterbox 改为新格式 keyframes:

```java
// 旧（删除）
lbClip.addProperty("enabled", true);
lbClip.addProperty("aspect_ratio", 2.35f);
lbClip.addProperty("fade_in", 0.5f);
lbClip.addProperty("fade_out", 0.5f);

// 新（替换为）
JsonArray kfs = new JsonArray();
JsonObject kf0 = new JsonObject(); kf0.addProperty("time", 0f); kf0.addProperty("aspect_ratio", 2.35f);
JsonObject kf1 = new JsonObject(); kf1.addProperty("time", 10f); kf1.addProperty("aspect_ratio", 2.35f);
kfs.add(kf0); kfs.add(kf1);
lbClip.add("keyframes", kfs);
```

**修复 3b** — ScriptParser.java 第 358 行加 fallback（安全兜底）:

```java
if (!obj.has("keyframes")) {
    float ratio = optFloat(obj, "aspect_ratio", 2.35f);
    float dur = requireFloat(obj, cp, "duration");
    LetterboxKeyframe k0 = new LetterboxKeyframe(0f, ratio);
    LetterboxKeyframe k1 = new LetterboxKeyframe(dur, ratio);
    clips.add(new LetterboxClip(
        requireFloat(obj, cp, "start_time"), dur, List.of(k0, k1)));
    continue;
}
JsonArray kfArr = obj.getAsJsonArray("keyframes");
```

**修复 3c** — init() firstInit (line 159-165) 和 menuBar onNewScript (line 230-237) 和 leftPanel onNewScript (line 444-451) 三处同改。

## 4. `/icinematics play <script>` 不带玩家参数报错

**文件**: `command/CinematicCommand.java` 第 114-118 行

**根因**: `EntityArgument.getPlayers(context, "players")` 在 `"players"` 节点不存在时（用户未带 `@a`）抛 `IllegalArgumentException`，但 catch 的是 `CommandSyntaxException`，异常未被捕获。

**修复**: 第 116 行 catch 改为:

```java
} catch (IllegalArgumentException | CommandSyntaxException e) {
    targets = server.getPlayerList().getPlayers();
}
```

同改 `stopScript` 第 138 行。

---

## 5. 下拉菜单文字被底层组件遮挡

**文件**: `editor/EditorScreen.java` 第 607-610 行

**根因**: ⑤ UI 树重构删除了 overlay 渲染时的 `RenderSystem.depthFunc(GL_ALWAYS)` 保护。MC 文字渲染器在其他渲染阶段绘制文字时覆盖了 overlay 内容，下拉菜单文本被其下方的组件文本穿透。

**修复**: 在 `renderOverlay` 调用前后恢复深度函数包裹，同时补回 import:

```java
// renderOverlay 前
RenderSystem.depthFunc(GL11.GL_ALWAYS);
rootComponent.renderOverlay(ctx);
RenderSystem.depthFunc(GL11.GL_LEQUAL);
```

**补回 import**:
```java
import com.mojang.blaze3d.systems.RenderSystem;
import org.lwjgl.opengl.GL11;
```
