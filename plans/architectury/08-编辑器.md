# 阶段 8：编辑器

迁移编辑器模块到 common 模块。编辑器是模组中最大的子系统（约 30 个文件），全部为纯 Vanilla Minecraft GUI 代码，无 Forge 依赖。

## 迁移步骤

### Step 1: 复制所有文件到 common

将 `editor/` 目录下所有文件复制到：
```
common/src/main/java/com/immersivecinematics/immersive_cinematics/editor/
```

将 `command/CinematicCommand.java` 复制到：
```
common/src/main/java/com/immersivecinematics/immersive_cinematics/command/
```

### Step 2: 确认引用完整性

编辑器文件引用了以下外部模块，需确认均已就绪：

| 引用目标 | 来源模块 | 状态 |
|---------|---------|------|
| `CameraManager` | camera/ | ✅ 阶段 2 |
| `CinematicController` | control/ | ✅ 阶段 2 |
| `CinematicKeyBindings` | control/ | ✅ 阶段 2 |
| `ScriptManager` | script/ | ✅ 阶段 3 |
| `ScriptParser` | script/ | ✅ 阶段 3 |
| `EditorBridge` | editor/ 自身 | ✅ |
| `EditorLogger` | editor/debug/ | ❌ 阶段 9 |
| `RawInputLogger` | editor/debug/ | ❌ 阶段 9 |

如果 `EditorScreen.java` 中对 `EditorLogger` 和 `RawInputLogger` 有编译期引用，创建临时桩或条件屏蔽。

### Step 3: 命令注册

`CinematicCommand.java` 是纯 Vanilla 的 Brigadier 命令，通过 `CommandRegistrationEvent.EVENT` 注册（已在阶段 7 中处理）。

### 测试清单
- [ ] Fabric 客户端启动正常
- [ ] Forge 客户端启动正常

- [ ] 编辑器界面正常渲染（菜单栏、左侧面板、预览、时间线）
- [ ] 新建脚本功能正常
- [ ] 打开已存在的 .cs 文件
- [ ] 保存脚本文件
- [ ] 播放控制（开始/暂停/停止）
- [ ] 时间线导航
- [ ] 关键帧编辑
- [ ] 触发器编辑面板正常
- [ ] 各种 widget 组件正常渲染和交互
- [ ] 命令 `/immersive_cinematics` 的各子命令正常

### 已知问题

1. **EditorLogger / RawInputLogger 缺失**：如果编译失败，创建空桩类：
   ```java
   // 临时桩，阶段 9 删除
   public class EditorLogger {
       public static void log(String msg) { }
       public static void log(String fmt, Object... args) { }
   }
   ```

2. **EDITOR_ENABLED 开关**：确认 `ImmersiveCinematics.EDITOR_ENABLED` 常量存在并设为 `true`

3. **EditorScreen 打开时机**：`CinematicKeyBindings.onClientTick()` 中检查 `mc.screen instanceof EditorScreen`，编辑器迁移后应能正常打开

## 编辑器测试方法

1. 启动游戏
2. 按 `F6` 打开编辑器
3. 点击 "New Script" 创建新脚本
4. 添加 Camera Clip
5. 添加关键帧并设置参数
6. 点击播放预览
7. 保存脚本并检查生成的文件
8. 重新打开编辑器加载已保存的脚本
