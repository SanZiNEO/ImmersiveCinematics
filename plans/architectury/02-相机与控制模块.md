# 阶段 2：相机与控制模块

## 目标

迁移相机系统（camera/）和控制器系统（control/）到 common 模块。这两个模块是模组的核心运行时，大部分是纯 Vanilla 代码。

这些文件不包含任何 `net.minecraftforge.*` 导入，仅引用 Minecraft Vanilla 类（`net.minecraft.*`），可直接编译。

**需要检查的内容**：
- CameraManager 是否引用了 Forge `ViewportEvent.ComputeCameraAngles`？
  - 从旧代码分析来看，CameraManager 只提供 `getProperties().getRoll()`，Forge ViewportEvent 的处理在 `ImmersiveCinematics.java` 的客户端事件中（ClientCameraEvents），不在 CameraManager 本身。
- 确认无其他隐含的平台依赖性

### Step 2: CinematicKeyBindings 改造

**当前状态**：`CinematicKeyBindings.java` 引用了 `RegisterKeyMappingsEvent` 来注册按键。

**改造方案**：

将按键注册拆分为两部分：
1. **common/**: 保留按键定义和业务逻辑
2. **forge/ + fabric/**: 平台各自注册按键

```java
// common/ - CinematicKeyBindings.java
public class CinematicKeyBindings {
    public static final KeyMapping SKIP_KEY = new KeyMapping(
        "key.immersive_cinematics.skip", GLFW.GLFW_KEY_C,
        "key.categories.immersive_cinematics"
    );
    public static final KeyMapping EDITOR_KEY = ImmersiveCinematics.EDITOR_ENABLED
        ? new KeyMapping("key.immersive_cinematics.editor", GLFW.GLFW_KEY_F6,
            "key.categories.immersive_cinematics")
        : null;

    // 保留所有业务逻辑：onClientTick(), getSkipHoldProgress(), notifyEditorClosed()
    // 删除 static void register(RegisterKeyMappingsEvent event)
}
```

```java
// forge/ - ForgeClientEvents.java（或直接放在 ImmersiveCinematicsForge 中）
// 使用 Architectury 的 ClientRegistry 注册按键
import dev.architectury.registry.client.keymappings.KeyMappingRegistry;

public static void registerKeyMappings() {
    KeyMappingRegistry.register(CinematicKeyBindings.SKIP_KEY);
    if (ImmersiveCinematics.EDITOR_ENABLED && CinematicKeyBindings.EDITOR_KEY != null) {
        KeyMappingRegistry.register(CinematicKeyBindings.EDITOR_KEY);
    }
}
```

## 已知问题/风险

1. **`EDITOR_ENABLED` 常量**：位于 `ImmersiveCinematics.java`，此阶段需确认其已正确迁移到 common
2. **无事件触发**：此阶段没有注册任何事件（ClientTick、ServerTick 等），所以相机管理器虽被加载但不活跃
3. **EditorScreen 引用**：`CinematicKeyBindings.onClientTick()` 中引用了 `EditorScreen`，此阶段 EditorScreen 尚未迁移。
   - **解决方案**：暂用条件编译或 try-catch 屏蔽 `ClassNotFoundException`
   - 或在 `onClientTick()` 中检查 `EDITOR_ENABLED` 标志，EditorScreen 仅在编辑器逻辑中引用
