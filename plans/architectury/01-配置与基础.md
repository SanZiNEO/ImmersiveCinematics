## 迁移步骤

### Step 1: Config 跨平台化

**当前状态**：`Config.java` 直接使用 `ForgeConfigSpec`，Forge-only。

**改造方案**：

```
Config.java (common/)
├── 保留所有静态字段 (skipHoldThresholdMs, showSkipHud 等)
├── 添加 ConfigProvider 接口
├── 添加 init() 方法，调用平台 provider 加载
├── 保留 setter 方法用于 ConfigScreen 调用
└── 删除所有 ForgeConfigSpec 引用

ForgeConfig.java (forge/)
└── 实现 ConfigProvider，使用 ForgeConfigSpec

FabricConfig.java (fabric/)
└── 实现 ConfigProvider，使用简单 JSON 配置
```

```java
// common/ - Config.java 骨架
public class Config {
    // 字段
    public static int skipHoldThresholdMs = 3000;
    public static boolean showSkipHud = true;
    public static int skipVoteRatio = 100;
    public static boolean debugLogging = false;
    public static int triggerPollIntervalLocation = 20;
    // ... 其他字段

    public interface ConfigProvider {
        void load();
        <T> T get(ConfigKey<T> key);
        <T> void set(ConfigKey<T> key, T value);
    }

    public static void init(ConfigProvider provider) {
        // 从 provider 加载所有值
    }
}
```

```java
// forge/ - ForgeConfig.java
public class ForgeConfig implements Config.ConfigProvider {
    private static final ForgeConfigSpec.Builder BUILDER = ...;
    // 使用 ForgeConfigSpec
}
```

```java
// fabric/ - FabricConfig.java
public class FabricConfig implements Config.ConfigProvider {
    // 使用 JSON 文件 + Properties
}
```

**测试要点**：
- [ ] 编译通过（common + fabric + forge）
- [ ] Fabric 下配置文件正确读写
- [ ] Forge 下配置文件正确读写
- [ ] ConfigScreen 可以正常显示和修改配置

### Step 2: 主类重构

**当前状态**：
- common/ `ImmersiveCinematics.java` 只有一个空 `init()` 方法
- forge/ `ImmersiveCinematicsForge.java` 使用 `@Mod` 注解
- fabric/ `ImmersiveCinematicsFabric.java` 使用 `ModInitializer`

**改造方案**（参考 `FTBQuests.java` 模式）：
- `common/ImmersiveCinematics.java`：保留 `MOD_ID`，`init()` 只做 config 初始化
- 各阶段逐步在 `init()` 中注册事件和初始化模块
- 客户端专用注册用 `EnvExecutor.runInEnv(Env.CLIENT, ...)` 包裹，不用客户端入口类硬耦合
- `forge/ImmersiveCinematicsForge.java`：在构造函数中调 `ImmersiveCinematics.init()`
- `fabric/ImmersiveCinematicsFabric.java`：在 `onInitialize()` 中调 `ImmersiveCinematics.init()`

此阶段 `init()` 只做：
```java
public static void init() {
    // 加载配置
    Config.init(platformConfigProvider);

    // 客户端专用注册（安全，不在服务端执行）
    EnvExecutor.runInEnv(Env.CLIENT, () -> () -> {
        // 后续阶段将在此添加客户端事件注册
    });
}
```

**主入口架构示意**（参考 FTB-Quests）：
```
forge/ImmersiveCinematicsForge.java
  └── new ImmersiveCinematics() → ImmersiveCinematics.init()

fabric/ImmersiveCinematicsFabric.java
  └── ImmersiveCinematics.init()

common/ImmersiveCinematics.java
  └── init()
      ├── Config.init(provider)
      ├── NetworkHandler.init()          → 后续阶段
      ├── ServerEventHandler.register()  → 后续阶段
      └── EnvExecutor.runInEnv(Env.CLIENT, () -> ClientEventHandler.register())
```
