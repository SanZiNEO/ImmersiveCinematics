# 阶段 3：脚本系统

## 目标

迁移完整的脚本系统（script/ 包）到 common 模块。脚本系统负责解析 `.cs` 脚本文件、管理时间线、播放相机/音频/信件等轨道。全部为纯 Vanilla 代码，可直接迁移。

## 迁移步骤

### Step 1: 直接复制到 common

将 `old/src/main/java/com/immersivecinematics/immersive_cinematics/script/` 下所有文件复制到：
```
common/src/main/java/com/immersivecinematics/immersive_cinematics/script/
```

**无需修改**，全部为纯 Vanilla 代码。

### Step 2: 验证 ScriptManager 的引用依赖

`ScriptManager.java` 中引用以下外部模块，需确认它们在本阶段可用：

| 引用目标 | 来源模块 | 状态 |
|---------|---------|------|
| `CameraManager` | camera/ | ✅ 阶段 2 已迁移 |
| `CinematicController` | control/ | ✅ 阶段 2 已迁移 |
| `ScriptParser` | script/ 自身 | ✅ 同模块 |
| `CinematicScript` | script/ 自身 | ✅ 同模块 |
| `TriggerDefinition` | script/ 自身 | ✅ 同模块 |
| `TrackPlayer` | script/ 自身 | ✅ 同模块 |
| `CameraTrackPlayer` | script/ 自身 | ✅ 同模块 |
| `AudioTrackPlayer` | script/ 自身 | ✅ 同模块 |
| `LetterboxTrackPlayer` | script/ 自身 | ✅ 同模块 |
| `EventTrackPlayer` | script/ 自身 | ✅ 同模块 |
| `ModEventTrackPlayer` | script/ 自身 | ✅ 同模块 |
| `TriggerEngine` | trigger/server/ | ❌ 尚未迁移 |
| `ScriptEventManager` | trigger/server/ | ❌ 尚未迁移 |
| `TriggerStateStore` | trigger/server/ | ❌ 尚未迁移 |
| `NetworkHandler` | trigger/network/ | ❌ 尚未迁移 |

`ScriptManager` 对未迁移模块的引用方式（需检查）：
- 如果通过接口或方法调用引用，编译会失败
- 如果只是声明类型或 import，需要在编译期可用

**风险处理**：如果编译失败，临时注释掉 `ScriptManager` 中对未迁移模块的引用，或者用 `// TODO Phase 6` 标记，等对应阶段再恢复。

### Step 3: 可选 — 编译时桩（如果必要）

如果 `ScriptManager` 对 `TriggerEngine` 等有强编译引用，创建一个临时桩接口：

```java
// common/ - 临时桩，Phase 6 删除
package com.immersivecinematics.immersive_cinematics.trigger.server;
public class TriggerEngine {
    public static final TriggerEngine INSTANCE = new TriggerEngine();
}
```

**仅在编译失败时使用**，否则直接迁移。
