## 迁移步骤

### Step 1: 迁移无依赖文件

直接复制以下文件到 `common/`：
- `TriggerRegistry.java`
- `TriggerRegistration.java`
- `ListenStrategy.java`
- `TriggerAction.java` + 所有 action 实现（除 PlaySoundAction）
- `Evaluators.java`
- `PlayerTriggerState.java`
- `TriggerStateStore.java`
- 所有 `trigger/client/` 文件

### Step 2: PlaySoundAction 改造

**当前状态**：使用 `ForgeRegistries.SOUND_EVENTS` 查找声音事件。

**改造方案**：使用 Vanilla `BuiltInRegistries.SOUND_EVENT` 替代。

```java
// 修改前
SoundEvent sound = (SoundEvent) ForgeRegistries.SOUND_EVENTS.getValue(soundId);

// 修改后
SoundEvent sound = BuiltInRegistries.SOUND_EVENT.get(soundId);
```

### Step 3: TriggerType 改造

**当前状态**：`TriggerType` 构造方法接收 `Set<Class<? extends Event>>`（Forge Event 类型），用于事件驱动触发器的调度。

**改造方案**：触发器类型使用字符串标识和泛型事件监听，不使用 Forge 的 Event 类。

```java
// TriggerType.java - 改造后
public class TriggerType {
    private final String id;
    private final ListenStrategy strategy;
    private final int pollInterval;
    private final BiPredicate<ServerPlayer, MinecraftServer> evaluator;
    // 移除 Set<Class<? extends Event>>，改用事件回调注册

    public TriggerType(String id, ListenStrategy strategy, int pollInterval,
                       BiPredicate<ServerPlayer, MinecraftServer> evaluator) {
        this.id = id;
        this.strategy = strategy;
        this.pollInterval = pollInterval;
        this.evaluator = evaluator;
    }
}
```

### Step 4: TriggerEngine 改造

**当前状态**：
- `onGameEvent(Event event, ServerPlayer player)` — 接收 Forge Event 对象
- 使用 `for (TriggerType type : eventDrivenTriggers)` + `type.matches(event)` 匹配

**改造方案**：

```java
// TriggerEngine.java - 改造后
public class TriggerEngine {
    // 不再接收 Forge Event 对象
    // 改为针对每种事件类型注册 Architectury 事件监听

    public void initialize() {
        // 为每个事件驱动的触发器类型注册 Architectury 事件
        // 例如:
        for (TriggerType type : eventDrivenTriggers) {
            registerEventDriven(type);
        }
    }

    private void registerEventDriven(TriggerType type) {
        switch (type.getId()) {
            case "advancement" -> PlayerEvent.ADVANCEMENT.register(player -> {
                // 调用 evaluator 检查条件
                checkAndFire(type, player);
            });
            case "entity_kill" -> EntityEvent.LIVING_DEATH.register((entity, source) -> {
                if (source.getEntity() instanceof ServerPlayer player) {
                    checkAndFire(type, player);
                }
            });
            // ... 更多事件类型
        }
    }

    // 轮询驱动触发器保留
    public void onServerTick(MinecraftServer server) {
        for (ServerPlayer player : server.getPlayerList().getPlayers()) {
            for (TriggerType type : pollingTriggers) {
                checkAndFire(type, player);
            }
        }
    }
}
```

**事件驱动触发器映射**（需要从旧 Forge 事件改为 Architectury 事件）：

| 触发器类型 | 旧 Forge 事件 | Architectury 事件 |
|-----------|-------------|-----------------|
| advancement | `AdvancementEvent.AdvancementEarnEvent` | `PlayerEvent.ADVANCEMENT`（或类似） |
| entity_kill | `LivingDeathEvent` | `EntityEvent.LIVING_DEATH` |
| entity_interact | `PlayerInteractEvent.EntityInteract` | `InteractionEvent.INTERACT_ENTITY` |
| dimension_change | `PlayerEvent.PlayerChangedDimensionEvent` | `PlayerEvent.CHANGE_DIMENSION` |
| login | `PlayerEvent.PlayerLoggedInEvent` | `PlayerEvent.PLAYER_JOIN` |
| item_craft | `PlayerEvent.ItemCraftedEvent` | `PlayerEvent.CRAFT_ITEM` |
| custom | 自定义事件 | 自定义实现 |
| item_use | `PlayerInteractEvent.RightClickItem` | `InteractionEvent.RIGHT_CLICK_ITEM` |
| item_consume | `LivingEntityUseItemEvent.Finish` | 可能需 Mixin |
| block_interact | `PlayerInteractEvent.RightClickBlock` / `LeftClickBlock` | `InteractionEvent.RIGHT_CLICK_BLOCK` / `LEFT_CLICK_BLOCK` |
| item_on_interact | 多个 Interact 事件组合 | 同上组合 |

> **注意**：某些 Architectury 事件可能不完全等价。在确认 Architectury API 提供的具体事件接口后，可能需要补充 Mixin。

### Step 5: ScriptEventManager 改造

**当前状态**：使用 `ServerLifecycleHooks.getCurrentServer()` 获取当前服务器实例。

**改造方案**：

```java
// ScriptEventManager.java - 改造后
public class ScriptEventManager {
    // 不再使用 ServerLifecycleHooks
    // 服务器引用通过以下方式传入：
    // 1. LifecycleEvent.SERVER_STARTED 回调中保存
    // 2. 方法参数传入（如 onServerTick(MinecraftServer server)）
    
    private MinecraftServer server;
    
    public void onServerStarted(MinecraftServer server) {
        this.server = server;
    }
    
    public void onServerTick(MinecraftServer server) {
        // 使用传入的 server 引用
        processPending(server);
    }
}
```

### Step 6: Trigger 注册重构

旧代码中，所有触发器类型在 `ImmersiveCinematics` 构造函数中注册：

```java
// 旧代码 - ImmersiveCinematics.java
private void registerTriggerTypes() {
    TriggerRegistry.register(new TriggerType("location", ListenStrategy.POLLING, 
        Config.triggerPollIntervalLocation, Evaluators::evaluateLocation, Set.of()));
    // ... 16 种触发器类型
}
```

改造后，这些注册逻辑移到 `common/ImmersiveCinematics.java` 的 `init()` 方法中，或独立的 `TriggerRegistration.java`：

```java
// TriggerRegistration.java - 改造后（无 Forge Event 参数）
public class TriggerRegistration {
    public static void registerAll() {
        TriggerRegistry.register(new TriggerType("location", ListenStrategy.POLLING,
            Config.triggerPollIntervalLocation, Evaluators::evaluateLocation));
        TriggerRegistry.register(new TriggerType("advancement", ListenStrategy.EVENT_DRIVEN,
            0, Evaluators::evaluateAdvancement));
        // ... 所有 16 种
    }
}
```

### 测试方法

1. 启动服务器，检查日志中 `TriggerRegistry` 的注册信息
2. 使用命令创建触发器（需 CinematicCommand 命令模块，Phase 9）
3. 或在开发环境中硬编码一个简单触发器进行测试

## 风险

1. **事件兼容性**：某些 Architectury 事件可能不如 Forge 事件详细（如 `PlayerInteractEvent.RightClickBlock` 的特定侧信息）。需验证 API 能力，必要时通过 Mixin 补充。
2. **Server 引用**：确保 `ScriptEventManager` 和 `TriggerEngine` 能正确获取 `MinecraftServer` 实例。
3. **Evaluator 追踪器**：`Evaluators.java` 中有多个静态追踪器（KillTracker, InteractTracker 等），它们不依赖 Forge，直接迁移即可，但需确认追踪数据的清理逻辑（在旧 main class 的 `onPlayerLogout` 中调用）。

## 额外步骤：玩家登出追踪器清理

旧代码中，玩家登出时清理追踪数据：
```java
// 旧 ImmersiveCinematics.java -> ServerForgeEvents.onPlayerLogout
Evaluators.KillTracker.clear(uuid);
Evaluators.InteractTracker.clear(uuid);
Evaluators.CraftTracker.clear(uuid);
// ...
```

改造后，通过 Architectury `PlayerEvent.PLAYER_QUIT` 事件注册清理逻辑：

```java
PlayerEvent.PLAYER_QUIT.register(player -> {
    UUID uuid = player.getUUID();
    Evaluators.KillTracker.clear(uuid);
    Evaluators.InteractTracker.clear(uuid);
    // ... 其他追踪器清理
});
```
