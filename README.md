## ✨ Features

### 🎥 Camera System
- **6DOF Camera Control** — position, yaw, pitch, roll, FOV, zoom, DOF
- **Frame-Callback Driven** — ReplayMod-style `System.nanoTime()` per render frame for sub-tick precision
- **Bézier Curve Paths** — cubic Bézier with extensible `PathStrategies` registry
- **Position Modes** — relative (`dx/dy/dz` from player) or absolute (`x/y/z` world coordinates)
- **Virtual Clock** — pause-aware timing; script freezes when game is paused

### 📜 JSON Script System (v3)
- **Multi-Track Timeline** — 5 track types: `CAMERA`, `LETTERBOX`, `AUDIO`, `EVENT`, `MOD_EVENT`
- **Keyframe Animation** — per-clip keyframe arrays with monotonic time validation
- **Clip Transitions** — `cut` (hard switch) and `crossfade` (smooth blend between clips)
- **Loop System** — configurable loop count, infinite duration support (`duration: -1`)
- **Strict Validation** — field-path-aware error reporting with `ScriptParseException`

### 🎨 Interpolation Engine
- **5 Curve Types** — `linear`, `smooth` (Hermite), `ease_in`, `ease_out`, `ease_in_out`
- **2 Interpolation Scopes** — `clip` (overall progress mapping) and `segment` (per-segment mapping)
- **2 Curve Composition Modes** — `override` (clip overrides script default) and `composed` (double smoothing)
- **3-Level Override** — script-level → clip-level → keyframe-level interpolation
- **NaN Protection** — all interpolated values sanitized with fallback to last valid frame

### 🖥️ Cinematic Overlay
- **Letterbox** — cinematic aspect ratio bars (2.35:1 default) with fade-in/out animation
- **Shader-Compatible** — uses `GuiGraphics.fill()` only, no OpenGL/FBO/shader conflicts
- **Hermite Easing** — smooth(t) = 3t² − 2t³ for natural bar transitions

## 🚀 Quick Start

### Installation

1. Install [Minecraft Forge 1.20.1](https://files.minecraftforge.net/net/minecraftforge/forge/index_1.20.1.html) (47.x+)
2. Download the latest release and place it in your `.minecraft/mods/` directory
3. Launch the game

### Commands

| Command | Description |
|---------|-------------|
| `/cinematic play <file>` | Load and play a script |
| `/cinematic stop` | Stop current playback |
| `/cinematic status` | Show playback status |

**File search order:**
1. Absolute path
2. Relative to game directory
3. `cinematics/` subdirectory
4. Auto-append `.json` extension

The `cinematics/` directory is auto-created on first use.

Create `cinematics/hello.json`:

      {

          {
            "start_time": 0.0,
            "duration": 10.0,
            "transition": "cut",
            "interpolation": "smooth",
            "interpolation_scope": "clip",
            "position_mode": "relative",
            "loop": false,
            "keyframes": [
              {
                "time": 0.0,
                "position": { "dx": 5, "dy": 2, "dz": 0 },
                "yaw": 90, "pitch": 5, "roll": 0,
                "fov": 70, "zoom": 1.0, "dof": 0
              },
              {
                "time": 10.0,
                "position": { "dx": -5, "dy": 3, "dz": 0 },
                "yaw": -90, "pitch": 15, "roll": 0,
                "fov": 70, "zoom": 1.0, "dof": 0
              }
            ]
          }
        ]
      },
      {
        "type": "LETTERBOX",
        "clips": [

            "duration": 10.0,

            "aspect_ratio": 2.35,
            "fade_in": 0.5,
            "fade_out": 0.5
          }

      }

## 📋 Script Format Reference

### Meta Section

```json
{
  "meta": {
    "id": "my_script",          // Required: ^[a-zA-Z0-9_]{1,32}$
    "name": "My Script",        // Required: max 50 chars
    "author": "Author",         // Required: max 30 chars
    "version": 3,               // Required: must be 3
    "description": "...",       // Optional
    "interpolation": "smooth",  // Optional: script-level default curve
    "curve_composition_mode": "override",  // Optional: override|composed
    // ... 15 runtime behavior flags (see table above)
  }
}
```

### Timeline & Tracks

```json
{
  "timeline": {
    "total_duration": 30.0,     // Positive=finite, negative=infinite, 0=invalid
    "tracks": [
      { "type": "CAMERA", "clips": [...] },
      { "type": "LETTERBOX", "clips": [...] },
      { "type": "AUDIO", "clips": [...] },
      { "type": "EVENT", "clips": [...] },
      { "type": "MOD_EVENT", "clips": [...] }
    ]
  }
}
```

### Camera Clip

```json
{
  "start_time": 0.0,
  "duration": 10.0,
  "transition": "cut",                    // cut | crossfade
  "crossfade_duration": 0.5,              // Only for crossfade
  "interpolation": "smooth",              // linear|smooth|ease_in|ease_out|ease_in_out
  "interpolation_scope": "clip",          // clip | segment
  "position_mode": "relative",            // relative (dx/dy/dz) | absolute (x/y/z)
  "loop": false,
  "loop_count": -1,                       // -1 = infinite
  "curve": {                              // Optional: Bézier path
    "type": "bezier",
    "control_points": [
      { "x": 1.0, "y": 0.0, "z": 0.5 },
      { "x": 2.0, "y": 0.0, "z": -0.5 }
    ]
  },
  "keyframes": [...]
}
```

### Camera Keyframe

```json
{
  "time": 0.0,
  "position": { "dx": 5, "dy": 2, "dz": 0 },  // or { "x": 100, "y": 64, "z": -200 }
  "yaw": 90,
  "pitch": 5,
  "roll": 0,
  "fov": 70,
  "zoom": 1.0,
  "dof": 0,
  "interpolation": "ease_in"   // Optional: per-keyframe override (SEGMENT scope only)
}
```

### Interpolation System

The interpolation engine supports a 3-level override hierarchy:

```
Script-level (meta.interpolation)
  └─ Clip-level (clip.interpolation) — overrides script default
       └─ Keyframe-level (keyframe.interpolation) — overrides clip (SEGMENT scope only)
```

**Interpolation Scopes:**
- **`clip`** — Curve mapping applies to overall clip progress; keyframe segments are linear within. Smooth at clip start/end only.
- **`segment`** — Curve mapping applies to each keyframe segment independently. Every segment has its own ease-in/ease-out.

**Curve Composition Modes:**
- **`override`** (default) — Clip/keyframe interpolation overrides script-level default. When clip is `linear` and script is `smooth`, uses `smooth`.
- **`composed`** — Mathematical composition: `adjustedT = clipCurve(scriptCurve(t))`. Produces double-smoothing effect.

## 🏗️ Architecture

```
┌─────────────────────────────────────────────────────────┐
│                    JSON Script (v3)                      │
└──────────────────────┬──────────────────────────────────┘
                       │ ScriptParser.parse()
                       ▼
┌─────────────────────────────────────────────────────────┐
│                  CinematicScript (POJO)                  │
│  ┌──────────┐  ┌──────────────────────────────────────┐ │
│  │ ScriptMeta│  │  Timeline → TimelineTrack[]          │ │
│  │  (15 flags│  │    ├─ CAMERA  → CameraClip[]        │ │
│  │   + interp│  │    ├─ LETTERBOX → LetterboxClip[]   │ │
│  │   + comp) │  │    ├─ AUDIO   → AudioClip[]         │ │
│  └──────────┘  │    ├─ EVENT   → EventClip[]          │ │
│                │    └─ MOD_EVENT → ModEventClip[]      │ │
│                └──────────────────────────────────────┘ │
└──────────────────────┬──────────────────────────────────┘
                       │ ScriptPlayer.start()
                       ▼
┌─────────────────────────────────────────────────────────┐
│              ScriptPlayer (Runtime Driver)               │
│  ┌─────────────────────────────────────────────────────┐│
│  │ TrackPlayer[] (per-track playback)                  ││
│  │  ├─ CameraTrackPlayer → KeyframeInterpolator        ││
│  │  ├─ LetterboxTrackPlayer → OverlayManager           ││
│  │  ├─ AudioTrackPlayer                                ││
│  │  └─ ModEventTrackPlayer                             ││
│  └─────────────────────────────────────────────────────┘│
└──────────────────────┬──────────────────────────────────┘
                       │ onRenderFrame() per frame
                       ▼
┌─────────────────────────────────────────────────────────┐
│              CameraManager (Singleton Bridge)            │
│  ┌──────────────────┐  ┌──────────────────┐            │
│  │ CameraProperties  │  │   CameraPath     │            │
│  │ (yaw/pitch/roll/  │  │ (position)       │            │
│  │  fov/zoom/dof)    │  │                  │            │
│  └────────┬─────────┘  └────────┬─────────┘            │
└───────────┼──────────────────────┼──────────────────────┘
            │                      │
            ▼                      ▼
┌─────────────────────────────────────────────────────────┐
│                    Mixin Layer (Read-Only)               │
│  CameraMixin.onSetup() → position, yaw, pitch           │
│  GameRendererMixin.onGetFov() → fov/zoom                │
│  ClientCameraEvents.onComputeCameraAngles() → roll      │
│  GameRendererMixin.onRenderItemInHand() → arm hiding    │
│  GameRendererMixin.onBobHurt/onBobView() → bob suppress │
└─────────────────────────────────────────────────────────┘
```

**Design Principles:**
- `CameraProperties` + `CameraPath` are pure POJOs — they know nothing about each other
- `CameraManager` is the **only bridge** between POJOs and the Mixin layer
- Mixin layer **only reads** from `CameraManager`, never writes
- Frame-callback driven: `onRenderFrame()` called from `CameraMixin.onSetup()` every render frame
